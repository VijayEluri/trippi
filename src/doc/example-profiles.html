Kowari Profile Example

The following responses in bold can be used to configure a Kowari triplestore for use with the Trippi Console. 

Enter the connector class name: org.trippi.impl.kowari.KowariConnector
Enter a short id for the new profile: ktest
Enter a label for the new profile: Kowari Test

Connector   : Kowari Connector
Class       : org.trippi.impl.kowari.KowariConnector
Description : This connector is written for Kowari v1.1.0 from CVS on Sept
              16th, patched with a memory leak fix. For more information on
              Kowari, visit http://www.kowari.org/

Parameter   : remote
Description : Whether the triplestore is available via rmi.
              This tells the connector to communicate with Kowari in remote or
              local mode.

Enter true or false: false

Parameter   : path
Description : The local path to the main triplestore directory.

Enter a value: /tmp/ktest

Parameter   : serverName
Description : The server name for rmi binding.

Enter a value: server1

Parameter   : modelName
Description : The name of the model to use.

Enter a value: ktest

Parameter   : poolInitialSize
Description : The initial size of the session pool used for queries.
              If one or greater, pooling will be used. If zero, all read/write
              access will be synchronized and no buffering will be done.

Enter a value: 3

Parameter   : poolMaxGrowth
Description : Maximum number of additional sessions the pool may add.
              If specified as -1, no limit will be placed on pool growth.

Enter a value: -1

Parameter   : poolSpareSessions
Description : The number of spare sessions to proactively make available.
              If unspecified, this defaults to 0, which means that additional
              sessions will only be created as needed. Note that if
              poolMaxGrowth is 0, the value of this parameter is
              inconsequential.

Enter a value (or [ENTER] for none): 0

Parameter   : readOnly
Description : Whether to connect in read-only mode.
              Read-only mode will prevent accidental changes to the
              triplestore.

Enter true or false: false

Parameter   : autoCreate
Description : Whether to create the model if it doesn't already exist.
              At startup, the model will be automatically created. In addition,
              an XML schema datatyped model named "xsd" will also be
              automatically created.

Enter true or false: true

Parameter   : autoTextIndex
Description : Whether to propogate adds/deletes to a fulltext model
              automatically.
              If true, the text model will be named modelName-fullText. Note
              that if this is true and autoCreate is true, the text model will
              also be created if it doesn't already exist.

Enter true or false: false

Parameter   : memoryBuffer
Description : Whether to use a memory buffer or a database buffer for write
              operations.
              The buffer is where triples are stored before they're actually
              written. Normally, a memory buffer will be sufficient. In certain
              server environments, a database buffer is the better option,
              offering higher capacity and persistence.

Enter true or false: true

Parameter   : autoFlushDormantSeconds
Description : Seconds of buffer inactivity that will trigger an auto-flush.
              If this threshold is reached, flushing will occur in the
              background, during which time the buffer is still available for
              writing.

Enter a value: 2

Parameter   : autoFlushBufferSize
Description : The size at which the buffer should be auto-flushed.
              If this threshold is reached, flushing will occur in the
              background, during which time the buffer is still available for
              writing.

Enter a value: 1000

Parameter   : bufferSafeCapacity
Description : The maximum size the buffer can reach before being forcibly
              flushed.
              If this threshold is reached, flushing will occur in the
              foreground and the buffer will be locked for writing until it is
              finished. This should be larger than autoFlushBufferSize.

Enter a value: 2000

Parameter   : bufferFlushBatchSize
Description : The number of updates to send to the triplestore at a time.
              This should be the same size as, or smaller than
              autoFlushBufferSize.

Enter a value: 1000

Profile created.  Type 'use ktest;' to use it.
